# 提案：添加文章类型管理功能

## 背景

当前系统中，文章的类型（type字段）是硬编码的整数值：
- 1 = 原创
- 2 = 转载  
- 3 = 翻译

这种设计存在以下问题：
1. **扩展性差**：如果需要新增文章类型（如"投稿"、"翻译修订"等），需要修改代码和数据库
2. **维护成本高**：类型定义散落在代码注释中，缺乏统一管理
3. **用户体验不佳**：前端只能展示固定的几个选项，无法动态配置

## 目标

创建一个独立的**文章类型管理模块**，实现：
- 文章类型的增删改查功能
- 支持动态配置文章类型，无需修改代码
- 与现有文章管理模块集成，替换硬编码的type字段
- 保持向后兼容，不影响已有数据

## 范围

### 包含
- 新建 `biz_article_type` 表
- 创建 ArticleType 实体、DTO、VO
- 实现完整的 CRUD 服务层
- 创建 REST API 控制器
- 修改 Article 实体关联文章类型
- 数据库迁移脚本

### 不包含
- 前端界面开发（后续迭代）
- 文章类型权限控制（使用默认权限）
- 文章类型统计数据展示（后续优化）

## 成功标准

- [ ] 可以通过 API 管理文章类型（增删改查）
- [ ] 文章创建/编辑时可以选择文章类型
- [ ] 文章列表可以正确显示文章类型名称
- [ ] 删除文章类型时，已关联的文章不受影响（类型ID保留）
- [ ] 所有接口通过 Swagger 文档可访问
